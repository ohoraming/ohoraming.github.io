카페/숙소/맛집 예약 웹페이지입니다

코로나19의 영향으로 공간마다 수용할 수 있는 인원이 제한되어 다양한 업종에서 예약 시스템의 필요성과 사용 빈도가 증가했습니다. 사업장마다 분산되어있던 예약 웹페이지를 연관된 업종끼리 모아, 통합 예약 서비스를 제공해 사용자에게 편의를 제공합니다.

  • 프로젝트 동작 영상입니다(클릭해 주세요↓)

  • 세부 페이지 소개

    회원 가입

    회원 가입 시스템으로 회원별 예약 리스트 관리를 합니다.

    회원별 email을 기본키로 설정한 쿼리를 만들어 MySQL을 사용해 회원 정보 데이터 베이스를 만들었습니다.

로그인

데이터 베이스에 저장된 이메일과 비밀번호의 일치 여부를 판단해 로그인을 진행합니다.

정보 확인 및 예약 시스템

공간에 대한 설명 확인 후 바로 예약이 가능하도록 구성하였습니다.

상품의 고유번호를 기본키로 설정해 상품 데이터 베이스를 만들었습니다.

예약 리스트 관리

예약을 하면 바로 로그인을 한 개인별로 리스트가 생성되며, 개별적인 수정/삭제가 가능합니다.

예약 리스트 데이터 베이스가 따로 존재하며, 사용자가 상품을 신규 예약할 때,
사용자의 이메일과 예약정보가 데이터 베이스에 추가됩니다.

상품 검색 기능

상품의 주소를 이용한 검색 기능입니다.

입력한 검색어와 상품 데이터 베이스의 주소 컬럼과의 일치를 확인하도록 쿼리를 작성했습니다.

상품 검색 결과

검색어와 상품의 주소가 일치하는 상품을 표시합니다.

관리자 페이지

관리자용 이메일와 비밀번호를 지정하였습니다.

로그인시 관리자용 이메일과 비밀번호를 데이터 베이스에서 일치 여부를 판단하고, 이메일을 이용한 조건문을 이용해 일반 사용자에서는 해당 페이지가 보이지 않도록 했습니다.

관리자 페이지-회원관리

관리자가 회원의 정보를 관리할 수 있습니다.

회원정보 데이터 베이스를 보여주는 페이지입니다.

관리자 페이지-상품관리

관리자가 상품을 등록, 수정, 삭제할 수 있습니다.

상품 데이터 베이스에 상품을 등록 후, 상품의 고유번호로 데이터 베이스를 조회, 수정, 삭제하도록 하였습니다.
