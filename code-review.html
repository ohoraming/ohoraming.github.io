이 글은 4월 우아한테크세미나 영상을 발표 자료와 함께 정리한 것입니다.

여러 오프라인 업체들을 온라인 업체가 대체해 나가고 있음

4차 산업 혁명의 중요한 키워드 중 하나, ‘VUCA’

  • 변동성(V)으로 인해 점점 기존의 방식으로는 문제를 해결할 수 없게 됨
  • 혼자가 아니라 여러 명이 협력해 문제를 해결하는 것의 가치가 높아지고 있으며,
    코드 리뷰는 변동성을 해결하는 데 도움을 주는 방법이 됨

전체 산업에서 IT의 비율이 10년만에 2배로 성장했고, 90%는 Digital Transformation (DT) 중임
Tech 이외의 영역에서 개발자 수요가 가파르게 증가하고 있음
미국은 보다 원활한 서비스를 공급하기 위해서는 3배(2020년 당시) 더 많은 개발자가 필요하다고 함
우리 나라의 경우, 미국의 1/4 수준의 개발자를 보유하고 있으므로 현 시점보다 12배 많은 개발자가 필요함

시장은 VUCA하고, 비즈니스는 빠른 혁신이 필요하므로, 개발에 있어서 SW를 더 자주 안정적으로 출시(DRFR)해야 함

(Clean Architecture 1장) 출시 때마다 개발자는 기하급수적으로 증가함

하지만, 출시 때마다 생산성은 일정 수치에 수렴함

처음에는 만든 모든 것이 산출물이라 기술부채가 없음
출시가 거듭될 수록 개발자는 기존 코드를 파악 및 수정하는데 대부분의 시간을 쓰고, 새로운 기능을 추가하는 생산적인 일을 하는데 시간을 쓰지 못하게 됨

Martin Fowler’s Design Stamina Hypothesis(블로그 참조)
좋은 설계와 나쁜 설계의 누적 생산성
설계 수익선을 지나 생기는 개발의 생산성 저하를 막아야 함

일반적인 공학의 경우, 설계와 빌드 중 빌드의 비용이 압도적임
건축물이 완공된 다음에는 변경할 수 있는 것이 거의 없으므로 유지보수 비용은 상대적으로 낮음

재생산이 가능하다는 것은 누가 빌드를 하든지 동일한 결과물이 나오는 것
SW빌드는 인건비가 발생하지 않아 거의 무료임
그래서 설계는 굉장히 중요하며, SW 엔지니어는 설계를 잘 하는 사람, 좋은 코드를 작성하는 사람임

새로운 코드를 작성하는 것은 20%
남의 코드를 읽고 변경하는 일이 80% 코드 이해에 10배 이상의 에너지가 소요됨
내 코드도 2주 뒤에는 남이 작성한 코드처럼 보임
코드가 좋은 가독성과 좋은 설계를 가져야 하는 이유임

일정에 쫓겨 빠르고 지저분하게 개발을 하는 것이 나중에 보면 결코 빠른 개발이 아님
SW의 경우, 비싸다고(비용이 많이 든다고) 품질이 좋은 것 아님(비정상적, 비직관적)
처음에는 비용이 많이 들어도, 수정할 때는 훨씬 비용을 절감할 수 있게 됨

테스트 커버리지를 높일 일정을 제공받고 싶다면, 개발자들이 원하는 것만 이야기해서는 소통이 힘듦
테스트 커버리지를 높였을 때 결함율이 감소하고, 변경 요청시 빠르게 처리가 가능해 출시 기간이 짧아질 수 있다고 말해야 비즈니스 측에서 원하는 대답이 됨

VUCA시대에 좋은 SW를 개발하는 가장 좋은 방법은 애자일임
하지만,
단순히 절차만 변경하고, TDD, refactoring, test 자동화, 단순한 설계 등 개발 역량에 대해서 다루지 않으면 애자일에 실패함
관련 지식과 경험은 학교가 아닌, 전문성을 갖춘 개발자가 공유를 해주는 방법이 유일함(이것이 SW의 장인정신, 애자일의 확장된 개념)

SW 장인정신은 공유 활동이 아주 중요함
공유 활동 중 코드 리뷰는 누구나 쉽게 접근할 수 있음
온라인에서 code로 댓글 놀이를 하면 코드 리뷰 하는 느낌을 받을 수 있음

(ref.Wikipedia)
코드 리뷰란?
한 명 또는 여러 사람이 주로 소스 코드 일부를 보고 읽는 방식으로, 프로그램을 확인하는 SW품질 보증 활동으로써 구현 혹은 구현을 잠시 중단하면서 시행함
피어 리뷰, 풀 리퀘스트, 머지 리퀘스트라고도 불림

코드 리뷰를 하며 양방향의 배움이 일어남
피어 간 또는 주니어 간의 코드 리뷰는 기술에만 한정된 하드 스킬을 배우게 됨
시니어는 주니어를 가르칠 때, 어떻게 소통을 해야 상대가 더 잘 받아들일 수 있는지에 대해 소프트 스킬을 배우게 됨 친절함, 공감능력, 가르치는 방법 등 리더십의 다양한 기법들을 배우게 됨
또한, 잘 하는 사람이 하는 것을 보며 동기 부여가 됨
